src.doubles_env

Entorno Gymnasium para batallas de dobles VGC (2v2).

Subclase de poke-env DoublesEnv que implementa:

  • embed_battle: observación detallada (ver src/features.py)
  • calc_reward: recompensa +1/-1 con bonus de HP

Envuelto en DoublesGymEnv para compatibilidad con SB3.

Nota: El espacio de acciones es MultiDiscrete([107, 107]) - una acción por cada pokemon activo. SB3 PPO soporta MultiDiscrete de forma nativa.

 37class _SuppressOpenTeamSheetFilter(logging.Filter):
 38    """
 39    Descarta el mensaje CRITICAL inofensivo de poke-env:
 40      "This format does not allow requesting open team sheets."
 41
 42    poke-env envia automáticamente /rejectopenteamsheets en cualquier formato
 43    VGC no-Bo3, pero algunos formatos (p.ej. gen9vgc2026regi no-Bo3) no permiten
 44    esa peticion y el servidor la rechaza. La batalla continua sin problemas;
 45    solo silenciamos el ruido en el log.

 48    def filter(self, record: logging.LogRecord) -> bool:
 49        return "open team sheets" not in record.getMessage().lower()

 52class MyDoublesEnv(DoublesEnv):
 53    """Entorno dobles con embedding propio y función de recompensa."""
 54
 55    def __init__(self, **kwargs):
 56        super().__init__(**kwargs)
 57        raw_obs = spaces.Box(
 58            low=np.float32(-10.0),
 59            high=np.float32(10.0),
 60            shape=(DOUBLES_OBS_SIZE,),
 61            dtype=np.float32,
 62        )
 63        self.observation_spaces = {
 64            agent: raw_obs for agent in self.possible_agents
 65        }
 66
 67    # ------------------------------------------------------------------
 68    # Metodos abstractos requeridos por PokeEnv
 69    # ------------------------------------------------------------------
 70
 71    def embed_battle(self, battle: AbstractBattle) -> npt.NDArray[np.float32]:
 72        assert isinstance(battle, DoubleBattle)
 73        return embed_doubles_battle(battle)
 74
 75    def calc_reward(self, battle: AbstractBattle) -> float:
 76        """
 77        Recompensa:
 78          +1.0 al ganar, -1.0 al perder.
 79          Pequenio bonus de HP intermedio.
 80        """
 81        assert isinstance(battle, DoubleBattle)
 82        if battle.finished:
 83            return 1.0 if battle.won else -1.0
 84
 85        # Bonus intermedio basado en ventaja de HP
 86        own_hp = sum(
 87            m.current_hp_fraction for m in battle.team.values()
 88        ) / max(len(battle.team), 1)
 89        opp_hp = sum(
 90            m.current_hp_fraction for m in battle.opponent_team.values()
 91        ) / max(len(battle.opponent_team), 1)
 92        return (own_hp - opp_hp) * 0.01

 95class DoublesGymEnv(Env):
 96    """
 97    Envoltorio Gymnasium sobre SingleAgentWrapper + MyDoublesEnv.
 98
 99    Aplana la observación (quita el action_mask) para compatibilidad
100    con SB3 PPO + MlpPolicy.
101
102    El espacio de acciones es MultiDiscrete([107, 107]).
103    SB3 maneja MultiDiscrete nativamente con la política "MlpPolicy".
104
105    Formatos soportados:
106      - "gen9randomdoublesbattle"  → equipos aleatorios, sin team_path, SIEMPRE funciona.
107      - "gen9vgc2025regg" (u otro VGC)  → requiere team_path con equipo VGC válido.
108
109    team_path puede ser:
110      - None        → formatos aleatorios (sin equipo)
111      - un .txt     → siempre el mismo equipo (mirror match fijo)
112      - una carpeta → equipo ALEATORIO por combate (cada batalla usa equipos
113                      distintos de tu lista; normalmente no-mirror)
114
115    Uso rápido (sin equipo):
116        env = DoublesGymEnv(port=8000)
117
118    Uso VGC con un equipo fijo:
119        env = DoublesGymEnv(port=8000, battle_format="gen9vgc2025regg",
120                            team_path="teams/vgc/equipo1.txt")
121
122    Uso VGC con equipos variados (carpeta):
123        env = DoublesGymEnv(port=8000, battle_format="gen9vgc2025regg",
124                            team_path="teams/vgc")
125
126    NOTA: gen9doublesou y similares no funcionan (poke-env solo soporta
127    teampreview para formatos VGC en SingleAgentWrapper).
128    """
129
130    metadata = {"render_modes": []}
131
132    def __init__(
133        self,
134        port: int = 8000,
135        battle_format: str = "gen9randomdoublesbattle",
136        log_level: int = 40,
137        team_path: str | None = None,
138        opponent: Player | None = None,
139        opponent_type: str = "random",   # "random" o "heuristic"
140    ):
141        from poke_env.player import MaxBasePowerPlayer, SimpleHeuristicsPlayer
142        from poke_env.ps_client.server_configuration import ServerConfiguration
143
144        from src.rl_wrapper import OpponentSamplingEnv
145        from src.teams import build_teambuilder
146
147        server_cfg = ServerConfiguration(
148            f"ws://localhost:{port}/showdown/websocket",
149            "https://play.pokemonshowdown.com/action.php?",
150        )
151
152        # team_path puede ser:
153        #   - un fichero .txt -> siempre el mismo equipo (ConstantTeambuilder)
154        #   - una carpeta     -> equipo aleatorio por combate (RandomTeamBuilder)
155        # Como los 2 agentes comparten el teambuilder, con una carpeta cada lado
156        # coge un equipo aleatorio distinto en cada batalla (no-mirror, variado).
157        team_builder = build_teambuilder(team_path)
158
159        # Factoria: crea un MyDoublesEnv listo. El wrapper la usa para crear el
160        # entorno y para RECONSTRUIRLO (nueva conexión/agentes) si se rompe.
161        def _make_poke_env():
162            pe = MyDoublesEnv(
163                battle_format=battle_format,
164                server_configuration=server_cfg,
165                log_level=log_level,
166                team=team_builder,
167                strict=False,        # acciones ilegales -> movimiento aleatorio
168                choose_on_teampreview=False,
169                # Temporizador del servidor como backstop ante cuelgues.
170                start_timer_on_battle_start=True,
171                # Si el nuevo desafío no se establece en este tiempo, poke-env
172                # lanza "Agent is not challenging". En localhost un desafío bueno
173                # tarda <1s; bajamos el defecto (60s) a 25s para que el wrapper
174                # detecte el fallo y reconstruya el entorno mucho más rápido.
175                challenge_timeout=25.0,
176            )
177            # Silenciar el CRITICAL inofensivo de open team sheets
178            ots = _SuppressOpenTeamSheetFilter()
179            pe.agent1.logger.addFilter(ots)
180            pe.agent2.logger.addFilter(ots)
181            return pe
182
183        # Oponente: "heuristic" -> SimpleHeuristicsPlayer (soporta dobles, sin
184        # conexión). "maxpower" -> MaxBasePowerPlayer (movimiento de más potencia).
185        # "random" -> None (muestreo de máscara). "self" -> se asigna el modelo
186        # después con set_opponent_model().
187        if opponent is None and opponent_type == "heuristic":
188            opponent = SimpleHeuristicsPlayer(
189                battle_format=battle_format,
190                start_listening=False,
191            )
192        elif opponent is None and opponent_type == "maxpower":
193            opponent = MaxBasePowerPlayer(
194                battle_format=battle_format,
195                start_listening=False,
196            )
197        self._opponent = opponent
198
199        # OpponentSamplingEnv reemplaza a SingleAgentWrapper (evita el bug de
200        # order_to_action de poke-env) y aporta recuperacion robusta (watchdog
201        # en step + reconstrucción del entorno si la conexión se rompe).
202        self._inner = OpponentSamplingEnv(_make_poke_env, opponent=self._opponent)
203
204        self.observation_space = self._inner.observation_space
205        self.action_space = self._inner.action_space
206
207    def reset(self, *, seed=None, options=None):
208        return self._inner.reset(seed=seed, options=options)
209
210    def step(self, action):
211        return self._inner.step(action)
212
213    def action_masks(self):
214        """Mascara de acciones legales para MaskablePPO."""
215        return self._inner.action_masks()
216
217    def set_opponent_model(self, model):
218        """Activa self-play: el rival jugara con la política de 'model'."""
219        self._inner.set_opponent_model(model)
220
221    def render(self):
222        return self._inner.render()
223
224    def close(self):
225        self._inner.close()
226
227    @property
228    def win_rate(self) -> float:
229        """Tasa de victorias del agente."""
230        agent = self._inner.poke_env.agent1
231        total = agent.n_finished_battles
232        return agent.n_won_battles / total if total > 0 else 0.0

302def build_doubles_obs(battle: DoubleBattle) -> npt.NDArray[np.float32]:
303    own_active = list(battle.active_pokemon) + [None, None]
304    opp_active = list(battle.opponent_active_pokemon) + [None, None]
305    own0, own1 = own_active[0], own_active[1]
306    opps = [opp_active[0], opp_active[1]]
307
308    feats: list[float] = []
309    feats += _global_features(battle)                      # 15
310    feats += _side_features(battle.side_conditions)        # 10
311    feats += _side_features(battle.opponent_side_conditions)  # 10
312    feats += _active_features(own0)                        # 40
313    feats += _active_features(own1)                        # 40
314    feats += _active_features(opps[0])                     # 40
315    feats += _active_features(opps[1])                     # 40
316
317    active_set = {id(m) for m in (own0, own1) if m is not None}
318    own_bench = [
319        m for m in battle.team.values() if id(m) not in active_set
320    ][:DOUBLES_BENCH]
321    for i in range(DOUBLES_BENCH):
322        feats += _reserve_features(own_bench[i] if i < len(own_bench) else None)
323    opp_active_set = {id(m) for m in opps if m is not None}
324    opp_bench = [
325        m for m in battle.opponent_team.values() if id(m) not in opp_active_set
326    ][:DOUBLES_BENCH]
327    for i in range(DOUBLES_BENCH):
328        feats += _reserve_features(opp_bench[i] if i < len(opp_bench) else None)
329
330    # Movimientos de cada activo propio, con efectividad vs los 2 rivales
331    feats += _moves_block(own0, opps, MOVE_FEATS_DOUBLES)  # 4*34
332    feats += _moves_block(own1, opps, MOVE_FEATS_DOUBLES)  # 4*34
333
334    assert len(feats) == DOUBLES_OBS_SIZE, f"{len(feats)} != {DOUBLES_OBS_SIZE}"
335    return np.array(feats, dtype=np.float32)

El espacio de acciones es MultiDiscrete([107, 107]). SB3 maneja MultiDiscrete nativamente con la política "MlpPolicy".

Formatos soportados:

  • "gen9randomdoublesbattle" → equipos aleatorios, sin team_path, SIEMPRE funciona.
  • "gen9vgc2025regg" (u otro VGC) → requiere team_path con equipo VGC válido.

team_path puede ser:

  • None → formatos aleatorios (sin equipo)
  • un .txt → siempre el mismo equipo (mirror match fijo)
  • una carpeta → equipo ALEATORIO por combate (cada batalla usa equipos distintos de tu lista; normalmente no-mirror)

Uso rápido (sin equipo): env = DoublesGymEnv(port=8000)

Uso VGC con un equipo fijo: env = DoublesGymEnv(port=8000, battle_format="gen9vgc2025regg", team_path="teams/vgc/equipo1.txt")

Uso VGC con equipos variados (carpeta): env = DoublesGymEnv(port=8000, battle_format="gen9vgc2025regg", team_path="teams/vgc")

NOTA: gen9doublesou y similares no funcionan (poke-env solo soporta teampreview para formatos VGC en SingleAgentWrapper).

Resets the environment to an initial internal state, returning an initial observation and info.

This method generates a new starting state often with some randomness to ensure that the agent explores the state space and learns a generalised policy about the environment. This randomness can be controlled with the seed parameter otherwise if the environment already has a random number generator and reset() is called with seed=None, the RNG is not reset.

Therefore, reset() should (in the typical use case) be called with a seed right after initialization and then never again.

For Custom environments, the first line of reset() should be super().reset(seed=seed) which implements the seeding correctly.

Changed in version v0.25: The return_info parameter was removed and now info is expected to be returned.

Args: seed (optional int): The seed that is used to initialize the environment's PRNG (np_random) and the read-only attribute np_random_seed. If the environment does not already have a PRNG and seed=None (the default option) is passed, a seed will be chosen from some source of entropy (e.g. timestamp or /dev/urandom). However, if the environment already has a PRNG and seed=None is passed, the PRNG will not be reset and the env's np_random_seed will not be altered. If you pass an integer, the PRNG will be reset even if it already exists. Usually, you want to pass an integer right after the environment has been initialized and then never again. Please refer to the minimal example above to see this paradigm in action. options (optional dict): Additional information to specify how the environment is reset (optional, depending on the specific environment)

Returns: observation (ObsType): Observation of the initial state. This will be an element of observation_space (typically a numpy array) and is analogous to the observation returned by step(). info (dictionary): This dictionary contains auxiliary information complementing observation. It should be analogous to the info returned by step().

Run one timestep of the environment's dynamics using the agent actions.

When the end of an episode is reached (terminated or truncated), it is necessary to call reset() to reset this environment's state for the next episode.

Changed in version 0.26: The Step API was changed removing done in favor of terminated and truncated to make it clearer to users when the environment had terminated or truncated which is critical for reinforcement learning bootstrapping algorithms.

Args: action (ActType): an action provided by the agent to update the environment state.

Returns: observation (ObsType): An element of the environment's observation_space as the next observation due to the agent actions. An example is a numpy array containing the positions and velocities of the pole in CartPole. reward (SupportsFloat): The reward as a result of taking the action. terminated (bool): Whether the agent reaches the terminal state (as defined under the MDP of the task) which can be positive or negative. An example is reaching the goal state or moving into the lava from the Sutton and Barto Gridworld. If true, the user needs to call reset(). truncated (bool): Whether the truncation condition outside the scope of the MDP is satisfied. Typically, this is a timelimit, but could also be used to indicate an agent physically going out of bounds. Can be used to end the episode prematurely before a terminal state is reached. If true, the user needs to call reset(). info (dict): Contains auxiliary diagnostic information (helpful for debugging, learning, and logging). This might, for instance, contain: metrics that describe the agent's performance state, variables that are hidden from observations, or individual reward terms that are combined to produce the total reward. In OpenAI Gym step() calls will return undefined results. This was removed in OpenAI Gym v26 in favor of terminated and truncated attributes. A done signal may be emitted for different reasons: Maybe the task underlying the environment was solved successfully, a certain timelimit was exceeded, or the physics simulation has entered an invalid state.

def render(self):

Compute the render frames as specified by render_mode during the initialization of the environment.

The environment's metadata render modes (env.metadata["render_modes"]) should contain the possible ways to implement the render modes. In addition, list versions for most render modes is achieved through gymnasium.make which automatically applies a wrapper to collect rendered frames.

Note: As the render_mode is known during __init__, the objects used to render the environment state should be initialised in __init__.

By convention, if the render_mode is:

  • None (default): no render is computed.
  • "human": The environment is continuously rendered in the current display or terminal, usually for human consumption. This rendering should occur during step() and render() doesn't need to be called. Returns None.
  • "rgb_array": Return a single frame representing the current state of the environment. A frame is a np.ndarray with shape (x, y, 3) representing RGB values for an x-by-y pixel image.
  • "ansi": Return a strings (str) or StringIO.StringIO containing a terminal-style text representation for each time step. The text can include newlines and ANSI escape sequences (e.g. for colors).
  • "rgb_array_list" and "ansi_list": List based version of render modes are possible (except Human) through the wrapper, gymnasium.wrappers.RenderCollection that is automatically applied during gymnasium.make(..., render_mode="rgb_array_list"). The frames collected are popped after render() is called or reset().

Note: Make sure that your class's metadata "render_modes" key includes the list of supported modes.

Changed in version 0.25.0: The render function was changed to no longer accept parameters, rather these parameters should be specified in the environment initialised, i.e., gymnasium.make("CartPole-v1", render_mode="human")
