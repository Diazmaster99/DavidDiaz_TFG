src.rl_wrapper

Wrapper Gymnasium que controla a un agente con RL y al oponente, con RECUPERACIÓN ROBUSTA ante los cuelgues conocidos de poke-env + servidor local.

Doble función: 1) Convierte el PokeEnv (PettingZoo, 2 agentes) en un entorno Gymnasium de un solo agente. agent1 = política RL; agent2 = oponente (aleatorio válido, heurístico, o self-play con una política). 2) Hace el entrenamiento RESILIENTE: - watchdog en step(): si un turno se queda colgado (desincronización poke-env/servidor en dobles), lo abandona en 'step_timeout' segundos en vez de esperar minutos al temporizador del servidor. - reconstrucción del entorno: si reset() o un turno fallan, se cierra la conexión rota y se crea un PokeEnv NUEVO (agentes/websocket limpios), que es lo único que recupera de un estado de conexión corrupto.

NOTA: el oponente se muestrea de la máscara de acciones (no se usa el SingleAgentWrapper de poke-env, que tiene un bug en order_to_action de dobles).

 54    def __init__(self, env_factory, opponent=None, step_timeout: float = 30.0):
 55        """
 56        :param env_factory: callable SIN argumentos que crea un PokeEnv listo
 57            (ya configurado, con sus filtros de log). Se usa para crear el
 58            entorno y para RECONSTRUIRLO si la conexión se rompe.
 59        :param opponent: Player opcional (p.ej. SimpleHeuristicsPlayer) para
 60            decidir las jugadas del rival. None -> aleatorio válido (o self-play
 61            si se asigna un modelo con set_opponent_model()).
 62        :param step_timeout: segundos máximos para un turno antes de considerar
 63            el combate atascado y reconstruir el entorno.
 64        """
 65        self._factory = env_factory
 66        self._env = env_factory()
 67        self._opponent = opponent
 68        self._opponent_model = None        # SELF-PLAY si se asigna
 69        self._step_timeout = step_timeout
 70        self._executor = ThreadPoolExecutor(max_workers=1)
 71
 72        a1 = self._env.agent1.username
 73        self.observation_space = self._env.observation_spaces[a1]["observation"]
 74        self.action_space = self._env.action_spaces[a1]
 75        self._is_doubles = isinstance(self.action_space, spaces.MultiDiscrete)
 76        if self._is_doubles:
 77            self._mask_len = int(self.action_space.nvec.sum())   # 214
 78        else:
 79            self._mask_len = int(self.action_space.n)            # 26
 80        self._last_mask = np.ones(self._mask_len, dtype=bool)

 95    def _force_finish_battles(self):
 96        """Marca como terminadas las batallas colgadas en los 2 agentes."""
 97        pe = self._env
 98        for agent in (getattr(pe, "agent1", None), getattr(pe, "agent2", None)):
 99            if agent is None:
100                continue
101            try:
102                for b in list(agent._battles.values()):
103                    if not b.finished:
104                        b._finished = True
105            except Exception:
106                pass
107        pe.battle1 = None
108        pe.battle2 = None
109
110    def _finish_lingering_battles(self):
111        """Marca como terminadas las batallas locales que hayan quedado sin
112        cerrar (p.ej. tras un turno abortado, o un final no detectado en uno de
113        los dos bandos en dobles). Así poke-env puede reiniciar sin lanzar
114        'Can not reset player's battles while they are still running', evitando
115        la costosa reconstrucción del entorno. No anula battle1/battle2: poke-env
116        los reasigna en su propio reset()."""
117        pe = self._env
118        for agent in (getattr(pe, "agent1", None), getattr(pe, "agent2", None)):
119            if agent is None:
120                continue
121            try:
122                for b in list(agent._battles.values()):
123                    if not b.finished:
124                        b._finished = True
125            except Exception:
126                pass
127
128    def _rebuild(self):
129        """Cierra el entorno roto y crea uno NUEVO (conexión/agentes limpios)."""
130        self._force_finish_battles()
131        try:
132            # IMPORTANTE: wait=False. El close() de poke-env, con su valor por
133            # defecto wait=True, hace self._challenge_task.result() SIN timeout;
134            # si el desafío está colgado (justo lo que provoca "Agent is not
135            # challenging"), ese .result() bloquea PARA SIEMPRE y congela todo el
136            # entrenamiento. Con wait=False se descarta la tarea colgada y se crea
137            # un entorno nuevo y limpio.
138            self._env.close(wait=False)
139        except Exception:
140            pass
141        # El worker del executor puede estar bloqueado en un step colgado:
142        # lo abandonamos y creamos uno nuevo.
143        try:
144            self._executor.shutdown(wait=False, cancel_futures=True)
145        except Exception:
146            pass
147        self._executor = ThreadPoolExecutor(max_workers=1)
148        time.sleep(2)
149        self._env = self._factory()

153    def reset(self, *, seed=None, options=None):
154        attempts = 4
155        for i in range(attempts):
156            try:
157                # Limpieza preventiva: cierra batallas locales sin terminar para
158                # evitar el OSError "battles still running" y la reconstrucción
159                # del entorno (que es lenta). Es inocua en el reset normal.
160                self._finish_lingering_battles()
161                obs, infos = self._env.reset(seed, options)
162                if self._opponent is not None and self._env.battle2 is not None:
163                    self._opponent.reset_battles()
164                    self._opponent._battles[
165                        self._env.battle2.battle_tag
166                    ] = self._env.battle2
167                a1 = self._env.agents[0]
168                self._last_mask = self._extract_mask(obs[a1])
169                return obs[a1]["observation"], infos[a1]
170            except (TimeoutError, OSError) as e:
171                # TimeoutError ("Agent is not challenging") y OSError ("battles
172                # still running") indican conexión/estado roto: reconstruimos.
173                if DEBUG:
174                    # Modo debug: sin reintentos; muestra el traceback real.
175                    print("[env.reset] MODO DEBUG: traceback real ->", flush=True)
176                    traceback.print_exc()
177                    raise
178                if i == attempts - 1:
179                    raise
180                print(
181                    f"[env.reset] fallo ({type(e).__name__}: {e}); reconstruyendo "
182                    f"entorno y reintentando ({i + 1}/{attempts - 1})...",
183                    flush=True,
184                )
185                self._rebuild()

187    def step(self, action):
188        pe = self._env
189        b1, b2 = pe.battle1, pe.battle2
190
191        # Guardia: si la batalla ya terminó, estado terminal limpio.
192        if (b1 is not None and b1.finished) or (b2 is not None and b2.finished):
193            reward = 0.0
194            if b1 is not None and b1.finished:
195                reward = 1.0 if b1.won else -1.0
196            pe.battle1 = None
197            pe.battle2 = None
198            self._last_mask = np.ones(self._mask_len, dtype=bool)
199            return self._zeros(), reward, True, False, {}
200
201        a1, a2 = pe.agents[0], pe.agents[1]
202        opp_action = self._opponent_action(b2)
203        actions = {a1: action, a2: opp_action}
204
205        # Ejecutar el turno con WATCHDOG: si tarda más de step_timeout, el
206        # combate está atascado -> abandonar y reconstruir (rápido, sin esperar
207        # al temporizador del servidor, que tarda minutos).
208        if DEBUG:
209            # Modo debug: sin watchdog ni captura -> traceback real del turno.
210            obs, rewards, terms, truncs, infos = pe.step(actions)
211        else:
212            try:
213                fut = self._executor.submit(pe.step, actions)
214                obs, rewards, terms, truncs, infos = fut.result(timeout=self._step_timeout)
215            except FuturesTimeoutError:
216                print(
217                    f"[env.step] turno atascado >{self._step_timeout:.0f}s; "
218                    f"abandonando combate y reconstruyendo entorno...",
219                    flush=True,
220                )
221                self._rebuild()
222                self._last_mask = np.ones(self._mask_len, dtype=bool)
223                return self._zeros(), 0.0, True, False, {}
224            except (AssertionError, ValueError, IndexError):
225                pe.battle1 = None
226                pe.battle2 = None
227                self._last_mask = np.ones(self._mask_len, dtype=bool)
228                return self._zeros(), 0.0, True, False, {}
229
230        a1_obs = obs[a1]
231        self._last_mask = self._extract_mask(a1_obs)
232        return (
233            a1_obs["observation"],
234            rewards[a1],
235            terms[a1],
236            truncs[a1],
237            infos[a1],
238        )

246    def _extract_mask(self, agent_obs) -> np.ndarray:
247        return self._sanitize_mask(agent_obs["action_mask"])
248
249    def _sanitize_mask(self, mask) -> np.ndarray:
250        mask = np.asarray(mask, dtype=bool)
251        if mask.shape[0] != self._mask_len:
252            return np.ones(self._mask_len, dtype=bool)
253        if self._is_doubles:
254            half = self._mask_len // 2
255            if not mask[:half].any():
256                mask[0] = True
257            if not mask[half:].any():
258                mask[half] = True
259        elif not mask.any():
260            mask[0] = True
261        return mask

263    def _zeros(self) -> np.ndarray:
264        return np.zeros(self.observation_space.shape, dtype=np.float32)

266    def _opponent_action(self, battle):
267        """
268        Acción del oponente, por prioridad:
269          1. SELF-PLAY (opponent_model) -> política RL.
270          2. HEURÍSTICO (Player) -> choose_move.
271          3. ALEATORIO -> muestreo de la máscara.
272        """
273        if self._opponent_model is not None:
274            try:
275                obs = self._env.embed_battle(battle)
276                mask = self._sanitize_mask(self._env.get_action_mask(battle))
277                action, _ = self._opponent_model.predict(
278                    obs, action_masks=mask, deterministic=False
279                )
280                action = np.asarray(action)
281                if self._is_doubles:
282                    return action.reshape(-1)[:2].astype(np.int64)
283                return np.int64(action.reshape(-1)[0])
284            except Exception:
285                if DEBUG:
286                    print("[opponent_model] error (se usaría aleatorio) ->", flush=True)
287                    traceback.print_exc()
288
289        if self._opponent is not None:
290            try:
291                from collections.abc import Awaitable
292                order = self._opponent.choose_move(battle)
293                if not isinstance(order, Awaitable):
294                    return self._env.order_to_action(
295                        order, battle, fake=False, strict=False
296                    )
297            except Exception:
298                if DEBUG:
299                    print("[opponent] error (se usaría aleatorio) ->", flush=True)
300                    traceback.print_exc()
301
302        return self._sample_from_mask(battle)
303
304    def _sample_from_mask(self, battle):
305        mask = self._env.get_action_mask(battle)
306        if self._is_doubles:
307            half = len(mask) // 2
308            valid0 = [i for i, m in enumerate(mask[:half]) if m] or [0]
309            valid1 = [i for i, m in enumerate(mask[half:]) if m] or [0]
310            return np.array(
311                [random.choice(valid0), random.choice(valid1)], dtype=np.int64
312            )
313        valid = [i for i, m in enumerate(mask) if m] or [0]
314        return np.int64(random.choice(valid))

The main Gymnasium class for implementing Reinforcement Learning Agents environments.

The class encapsulates an environment with arbitrary behind-the-scenes dynamics through the step() and reset() functions. An environment can be partially or fully observed by single agents. For multi-agent environments, see PettingZoo.

The main API methods that users of this class need to know are:

  • step() - Updates an environment with actions returning the next agent observation, the reward for taking that actions, if the environment has terminated or truncated due to the latest action and information from the environment about the step, i.e. metrics, debug info.
  • reset() - Resets the environment to an initial state, required before calling step. Returns the first agent observation for an episode and information, i.e. metrics, debug info.
  • render() - Renders the environments to help visualise what the agent see, examples modes are "human", "rgb_array", "ansi" for text.
  • close() - Closes the environment, important when external software is used, i.e. pygame for rendering, databases

Environments have additional attributes for users to understand the implementation

  • action_space - The Space object corresponding to valid actions, all valid actions should be contained within the space.
  • observation_space - The Space object corresponding to valid observations, all valid observations should be contained within the space.
  • spec - An environment spec that contains the information used to initialize the environment from gymnasium.make()
  • metadata - The metadata of the environment, e.g. {"render_modes": ["rgb_array", "human"], "render_fps": 30}. For Jax or Torch, this can be indicated to users with "jax"=True or "torch"=True.
  • np_random - The random number generator for the environment. This is automatically assigned during super().reset(seed=seed) and when assessing np_random.

seealso For modifying or extending environments use the gymnasium.Wrapper class.

Note: To get reproducible sampling of actions, a seed can be set with env.action_space.seed(123).

Note: For strict type checking (e.g. mypy or pyright), Env is a generic class with two parameterized types: ObsType and ActType. The ObsType and ActType are the expected types of the observations and actions used in reset() and step(). The environment's observation_space and action_space should have type Space[ObsType] and Space[ActType], see a space's implementation to find its parameterized type.

Resets the environment to an initial internal state, returning an initial observation and info.

This method generates a new starting state often with some randomness to ensure that the agent explores the state space and learns a generalised policy about the environment. This randomness can be controlled with the seed parameter otherwise if the environment already has a random number generator and reset() is called with seed=None, the RNG is not reset.

Therefore, reset() should (in the typical use case) be called with a seed right after initialization and then never again.

For Custom environments, the first line of reset() should be super().reset(seed=seed) which implements the seeding correctly.

Changed in version v0.25: The return_info parameter was removed and now info is expected to be returned.

Args: seed (optional int): The seed that is used to initialize the environment's PRNG (np_random) and the read-only attribute np_random_seed. If the environment does not already have a PRNG and seed=None (the default option) is passed, a seed will be chosen from some source of entropy (e.g. timestamp or /dev/urandom). However, if the environment already has a PRNG and seed=None is passed, the PRNG will not be reset and the env's np_random_seed will not be altered. If you pass an integer, the PRNG will be reset even if it already exists. Usually, you want to pass an integer right after the environment has been initialized and then never again. Please refer to the minimal example above to see this paradigm in action. options (optional dict): Additional information to specify how the environment is reset (optional, depending on the specific environment)

Returns: observation (ObsType): Observation of the initial state. This will be an element of observation_space (typically a numpy array) and is analogous to the observation returned by step(). info (dictionary): This dictionary contains auxiliary information complementing observation. It should be analogous to the info returned by step().

Run one timestep of the environment's dynamics using the agent actions.

When the end of an episode is reached (terminated or truncated), it is necessary to call reset() to reset this environment's state for the next episode.

Changed in version 0.26: The Step API was changed removing done in favor of terminated and truncated to make it clearer to users when the environment had terminated or truncated which is critical for reinforcement learning bootstrapping algorithms.

Args: action (ActType): an action provided by the agent to update the environment state.

Returns: observation (ObsType): An element of the environment's observation_space as the next observation due to the agent actions. An example is a numpy array containing the positions and velocities of the pole in CartPole. reward (SupportsFloat): The reward as a result of taking the action. terminated (bool): Whether the agent reaches the terminal state (as defined under the MDP of the task) which can be positive or negative. An example is reaching the goal state or moving into the lava from the Sutton and Barto Gridworld. If true, the user needs to call reset(). truncated (bool): Whether the truncation condition outside the scope of the MDP is satisfied. Typically, this is a timelimit, but could also be used to indicate an agent physically going out of bounds. Can be used to end the episode prematurely before a terminal state is reached. If true, the user needs to call reset(). info (dict): Contains auxiliary diagnostic information (helpful for debugging, learning, and logging). This might, for instance, contain: metrics that describe the agent's performance state, variables that are hidden from observations, or individual reward terms that are combined to produce the total reward. In OpenAI Gym step() calls will return undefined results. This was removed in OpenAI Gym v26 in favor of terminated and truncated attributes. A done signal may be emitted for different reasons: Maybe the task underlying the environment was solved successfully, a certain timelimit was exceeded, or the physics simulation has entered an invalid state.

Compute the render frames as specified by render_mode during the initialization of the environment.

The environment's metadata render modes (env.metadata["render_modes"]) should contain the possible ways to implement the render modes. In addition, list versions for most render modes is achieved through gymnasium.make which automatically applies a wrapper to collect rendered frames.

Note: As the render_mode is known during __init__, the objects used to render the environment state should be initialised in __init__.

By convention, if the render_mode is:

  • None (default): no render is computed.
  • "human": The environment is continuously rendered in the current display or terminal, usually for human consumption. This rendering should occur during step() and render() doesn't need to be called. Returns None.
  • "rgb_array": Return a single frame representing the current state of the environment. A frame is a np.ndarray with shape (x, y, 3) representing RGB values for an x-by-y pixel image.
  • "ansi": Return a strings (str) or StringIO.StringIO containing a terminal-style text representation for each time step. The text can include newlines and ANSI escape sequences (e.g. for colors).
  • "rgb_array_list" and "ansi_list": List based version of render modes are possible (except Human) through the wrapper, gymnasium.wrappers.RenderCollection that is automatically applied during gymnasium.make(..., render_mode="rgb_array_list"). The frames collected are popped after render() is called or reset().

Note: Make sure that your class's metadata "render_modes" key includes the list of supported modes.

Changed in version 0.25.0: The render function was changed to no longer accept parameters, rather these parameters should be specified in the environment initialised, i.e., gymnasium.make("CartPole-v1", render_mode="human")
